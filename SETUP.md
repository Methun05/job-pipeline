# Job Pipeline — Setup Guide
Complete in ~20 minutes. All free tiers.

---

## Step 1 — Supabase (5 min)

1. Go to https://supabase.com → New project
2. Name it `job-pipeline`, choose a region close to you (e.g. `eu-west-1`)
3. After project creates: **Settings → API**
   - Copy `Project URL` → `SUPABASE_URL`
   - Copy `anon/public` key → `SUPABASE_ANON_KEY`
   - Copy `service_role` key → `SUPABASE_SERVICE_KEY` (**keep this secret**)
4. Go to **SQL Editor → New Query**
5. Paste the entire contents of `supabase/migrations/001_initial_schema.sql`
6. Click **Run** — should say "Success"

---

## Step 2 — Groq API (2 min)

1. Go to https://console.groq.com → Sign up (free)
2. Click **API Keys → Create API Key**
3. Copy the key → `GROQ_API_KEY`

---

## Step 3 — Apollo.io (3 min)

1. Go to https://apollo.io → Sign up (free)
2. Go to **Settings → Integrations → API**
3. Copy your API key → `APOLLO_API_KEY`
4. Free tier gives you: People Search (unlimited) + 100 email credits/month

---

## Step 4 — CryptoRank (2 min)

1. Go to https://cryptorank.io/public-api → Sign up
2. Generate API key → `CRYPTORANK_API_KEY`

---

## Step 5 — Create GitHub Repository (5 min)

1. Go to https://github.com/new
2. Name: `job-pipeline` | Private | No README
3. Click **Create repository**
4. In your terminal, run these commands:

```bash
cd ~/job-pipeline
git init
git add .
git commit -m "Initial pipeline setup"
git remote add origin https://github.com/YOUR_USERNAME/job-pipeline.git
git push -u origin main
```

5. Go to repo **Settings → Secrets and variables → Actions → New repository secret**
   Add these secrets (one by one):
   - `SUPABASE_URL`
   - `SUPABASE_SERVICE_KEY`
   - `GROQ_API_KEY`
   - `APOLLO_API_KEY`
   - `CRYPTORANK_API_KEY`

---

## Step 6 — Vercel Dashboard (3 min)

1. Go to https://vercel.com → New Project → Import from GitHub
2. Select your `job-pipeline` repo
3. Set **Root Directory** to `dashboard`
4. Add Environment Variables:
   ```
   NEXT_PUBLIC_SUPABASE_URL      = (your Supabase URL)
   NEXT_PUBLIC_SUPABASE_ANON_KEY = (your anon key)
   SUPABASE_SERVICE_KEY          = (your service role key)
   APOLLO_API_KEY                = (your Apollo key)
   DASHBOARD_PASSWORD            = (pick a password for your girlfriend)
   ```
5. Click **Deploy**

---

## Step 7 — Custom Domain (2 min)

1. In Vercel → your project → **Settings → Domains**
2. Add `tracker.methun.design`
3. In your DNS provider (wherever methun.design is managed):
   - Add CNAME: `tracker` → `cname.vercel-dns.com`
4. Wait 1-5 minutes for DNS to propagate

---

## Step 8 — Test the Pipeline

1. Go to your GitHub repo → **Actions** tab
2. Click **Daily Job Pipeline** → **Run workflow**
3. Watch the logs — should complete in ~5 minutes
4. Open https://tracker.methun.design → enter your password → see results

---

## Local development (optional)

```bash
# Python pipeline
cd ~/job-pipeline
pip install -r requirements.txt
cp .env.example .env   # fill in your keys
python -m pipeline.main

# Dashboard
cd ~/job-pipeline/dashboard
npm install
cp ../.env.example .env.local   # fill in your keys
npm run dev
# Open http://localhost:3000
```

---

## What runs automatically

- Pipeline: daily at 8:00 AM IST (GitHub Actions cron)
- Dashboard: always live at tracker.methun.design
- Follow-ups: generated by pipeline when contact is 7+ days inactive
- Apollo credits: monitored daily, dashboard warns when < 30

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Pipeline fails | Check GitHub Actions logs → Errors tab |
| Dashboard shows "Pipeline hasn't run" | Check Actions tab, trigger manually |
| Apollo credits at 0 | Entire Apollo API stops — wait for monthly reset or upgrade |
| Supabase quota | Free tier: 500MB DB, 2GB bandwidth — more than enough |
